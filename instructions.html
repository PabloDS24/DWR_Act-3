Indicaciones para realizar la actividad

Actividad grupal

Durante la actividad vas a atajar el desarrollo, junto a un grupo de tus compañeros de clase, de una aplicación web haciendo uso de las herramientas aprendidas. También tendrán que hacer uso de repositorio de código (Github, Bitbucket o similar) para el trabajo colaborativo. Este repositorio le servirá al profesor para corroborar la aportación de cada uno de los integrantes del equipo al proyecto.

Debe tener al menos cuatro pantallas.

Una de esas pantallas debe ser un formulario de login (este login no es funcional 100 %, simplemente debe simular el proceso). Debe persistir en session storage el correo del usuario y usarlo en alguna de las pantallas restantes.

Listado de funcionalidades que se deben incorporar
  • Debe desarrollar una funcionalidad de logout que limpie la información del session storage y lleve a la pantalla de login.
  • Debe validar que el valor en el campo «correo» es válido.
  • Debe tener al menos dos formularios diferentes al login. El 70 % de los campos deben ser requeridos y validados.
  • Debe persistir en local storage la información de los formularios. Esta información debe ser usada en cualquier otra pantalla.
  • Debe usar al menos cinco componentes de Bootstrap.
  • Puede usar flexbox o la malla para alinear los elementos.
  • La aplicación debe ser navegable entre las pantallas.
  • De consumir un API gratis público y utilizarlo donde se prefiera.
